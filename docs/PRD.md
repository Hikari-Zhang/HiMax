## 1. 产品概述

### 1.1 产品定位

**HiMax** — 一款**开源的跨平台视频下载桌面应用**，提供类似 macOS Downie 的体验，但面向 Windows 和 macOS 双平台。用户只需粘贴视频链接，即可快速下载全球主流视频平台的内容，支持画质选择、格式转换、批量下载等高级功能。

### 1.2 目标用户

| 用户类型 | 典型场景 |
|---------|---------|
| 视频爱好者 | 收藏喜欢的视频到本地离线观看 |
| 内容创作者 | 下载素材用于二次创作（合规范围内） |
| 学习者 | 下载教学视频离线学习 |
| 开发者 | 对开源项目感兴趣，想要参与贡献 |

### 1.3 核心价值主张

- **极简体验**：粘贴链接 → 选择画质 → 一键下载，三步搞定
- **全面覆盖**：基于 yt-dlp，支持 1000+ 网站（国内外全覆盖）
- **开源免费**：代码完全开源，社区驱动
- **跨平台**：Windows + macOS 原生体验

### 1.4 技术栈

| 层面 | 选型 |
|------|------|
| 桌面框架 | Electron |
| 前端 | React + TypeScript |
| 样式 | Tailwind CSS + shadcn/ui |
| 状态管理 | Zustand |
| 视频解析/下载 | yt-dlp（子进程调用） |
| 媒体处理 | FFmpeg |
| 数据库 | better-sqlite3 |
| 国际化 | i18next（中/英双语） |
| 打包 | electron-builder |

### 2.2 模块 1：下载核心

#### F1.1 URL 输入与解析

| 项目 | 说明 |
|------|------|
| **功能描述** | 用户输入视频 URL 后，自动解析视频信息并展示 |
| **触发方式** | ① 手动粘贴/输入 URL ② 剪贴板自动监听 ③ 拖拽链接到窗口 |
| **输入** | 视频页面 URL / 播放列表 URL / 频道 URL |
| **输出** | 视频元信息（标题、缩略图、时长、作者、可用格式列表） |

**详细需求**：

- **URL 输入栏**：位于界面顶部，常驻显示，支持：
  - 手动输入/粘贴 URL
  - 右侧「粘贴」按钮一键从剪贴板读取
  - 右侧「解析」按钮触发解析
  - 回车键触发解析
  - 支持一次粘贴多行 URL（以换行分隔），自动识别为批量任务

- **剪贴板自动监听**：
  - 可在设置中开启/关闭（默认关闭）
  - 开启后，每秒检测剪贴板内容变化
  - 检测到视频 URL 时，弹出浮窗确认「检测到视频链接，是否下载？」
  - 浮窗展示视频标题缩略图预览，提供「下载」和「忽略」按钮
  - 支持配置白名单/黑名单域名

- **拖拽支持**：
  - 支持从浏览器拖拽链接到应用窗口
  - 拖入时显示高亮投放区域视觉反馈
  - 放下后自动触发解析

- **解析过程**：
  - 显示解析中 loading 状态
  - 超时设置 30 秒，超时后提示用户重试
  - 解析失败时显示错误原因和建议（如「此网站可能需要 Cookie 登录」）

#### F1.2 视频信息预览

| 项目 | 说明 |
|------|------|
| **功能描述** | 解析完成后展示视频详情，供用户选择下载参数 |
| **触发条件** | URL 解析成功 |

**展示信息**：

```
┌───────────────────────────────────────────────────┐
│ ┌─────────┐                                       │
│ │         │  标题：视频标题                         │
│ │  缩略图  │  作者：上传者名称                       │
│ │         │  时长：12:34                           │
│ │         │  来源：youtube.com                     │
│ └─────────┘                                       │
│                                                   │
│  画质：[▼ 1080p / 720p / 480p / 360p / 最佳 ]     │
│  格式：[▼ MP4 / MKV / WebM / 仅音频(MP3/M4A) ]    │
│  字幕：[▼ 无 / 中文 / 英文 / 中英双语 ]            │
│  保存到：[📁 D:\Videos\Downloads ] [更改...]       │
│  文件名：[可编辑的文件名输入框        ]             │
│                                                   │
│              [ 取消 ]  [ ⬇️ 开始下载 ]              │
└───────────────────────────────────────────────────┘
```

**详细需求**：

- **画质选择器**：
  - 下拉列表展示所有可用分辨率
  - 每个选项显示分辨率 + 预估文件大小 + 编码格式
  - 格式示例：`1080p (H.264) ~350MB` / `4K (VP9) ~1.2GB`
  - 默认选中「最佳画质」或用户设置的默认偏好
  - 如果某个分辨率需要合并音视频流，显示提示标记

- **格式选择器**：
  - MP4（最通用）、MKV（支持更多特性）、WebM
  - 仅音频模式：MP3、M4A、FLAC、WAV
  - 选择「仅音频」时，画质选择器切换为音质选择（128k/192k/320k/最佳）

- **字幕选择器**：
  - 列出所有可用字幕语言
  - 支持多选（同时下载多种语言字幕）
  - 字幕格式选择：SRT / ASS / VTT
  - 支持嵌入字幕到视频文件（硬字幕/软字幕）

- **保存路径**：
  - 显示当前目标文件夹路径
  - 「更改」按钮打开系统文件夹选择对话框
  - 默认使用设置中的默认下载目录
  - 记住最近使用的 5 个目录，方便快速切换

- **文件名编辑**：
  - 默认使用视频原标题
  - 支持直接编辑修改
  - 自动过滤非法文件名字符
  - 文件名冲突时提示：覆盖 / 重命名(添加序号) / 取消

#### F1.3 批量下载

| 项目 | 说明 |
|------|------|
| **功能描述** | 支持播放列表、频道、多链接的批量下载 |

**场景 A — 播放列表/合集下载**：

- 识别到播放列表 URL 后，先展示列表概览：
  - 播放列表名称、总视频数、总时长估算
  - 以列表形式展示所有视频（勾选框 + 缩略图 + 标题 + 时长）
  - 「全选 / 全不选 / 反选」快捷操作
  - 统一设置画质/格式，或单独为某个视频设置
- 支持选择部分视频下载（不必下载整个列表）

**场景 B — 批量粘贴多链接**：

- URL 输入框支持多行粘贴
- 自动拆分、逐一解析
- 以列表形式展示所有解析结果
- 统一或单独设置下载参数

**场景 C — 频道/UP 主页面**：

- 输入频道/UP 主主页 URL
- 展示频道信息（名称、头像、视频总数）
- 加载最近 N 个视频供选择（支持分页加载更多）
- 支持按时间范围筛选

### 2.3 模块 2：下载任务管理

#### F2.1 下载队列

| 项目 | 说明 |
|------|------|
| **功能描述** | 管理所有下载任务的生命周期 |

**任务状态流转**：

```
[排队中] ──→ [解析中] ──→ [下载中] ──→ [后处理中] ──→ [已完成]
   │            │            │              │
   │            ▼            ▼              ▼
   │        [解析失败]   [已暂停]       [处理失败]
   │                       │
   │                       ▼
   └──────────────────→ [已取消]
```

**任务状态说明**：

| 状态 | 图标 | 说明 |
|------|------|------|
| 排队中 | 🕐 | 等待下载槽位 |
| 解析中 | 🔍 | 正在解析视频信息 |
| 下载中 | ⬇️ | 正在下载，显示进度条/速度/ETA |
| 后处理中 | ⚙️ | 合并音视频流/转码/嵌入字幕 |
| 已完成 | ✅ | 下载完成 |
| 已暂停 | ⏸️ | 用户手动暂停 |
| 已取消 | ❌ | 用户取消 |
| 解析失败 | ⚠️ | URL 无法解析 |
| 下载失败 | ❌ | 下载过程中出错 |

#### F2.2 任务卡片

每个下载任务以卡片形式展示：

```
┌───────────────────────────────────────────────────────┐
│ ┌────────┐                                            │
│ │        │  📺 视频标题（最多两行，超出省略...）        │
│ │ 缩略图  │  👤 作者名 · 🌐 youtube.com · ⏱️ 12:34    │
│ │        │  📦 1080p MP4 · 📁 D:\Videos               │
│ └────────┘                                            │
│                                                       │
│  ████████████████████░░░░░░░░  78.5%                  │
│  ⬇️ 5.2 MB/s  │  📊 267/340 MB  │  ⏳ ETA 0:14       │
│                                                       │
│  [ ⏸️ 暂停 ] [ ❌ 取消 ] [ 📂 打开文件夹 ] [ ⋮ 更多 ] │
└───────────────────────────────────────────────────────┘
```

**「更多」菜单**：
- 复制下载链接
- 重新下载
- 在文件管理器中显示
- 查看详细日志
- 删除任务（仅删记录 / 同时删文件）

#### F2.3 并发与调度控制

| 配置项 | 默认值 | 范围 |
|--------|--------|------|
| 最大并发下载数 | 3 | 1-10 |
| 单任务最大连接数 | 8 | 1-32 |
| 全局限速 | 不限 | 0（不限）/ 自定义 KB/s 或 MB/s |
| 单任务限速 | 不限 | 同上 |

#### F2.4 优先级与排序

- 下载队列支持拖拽排序，改变下载顺序
- 右键菜单「置顶」/「移到底部」
- 排队中的任务才可调整顺序

#### F2.5 定时下载

- 支持设置下载时间窗口（如 01:00-06:00 自动下载）
- 在时间窗口外的任务自动暂停，进入窗口后自动恢复
- 用途：利用夜间闲时带宽

### 2.5 模块 4：附加功能

#### F4.1 下载历史

| 项目 | 说明 |
|------|------|
| **功能描述** | 记录所有下载任务的历史，支持搜索和筛选 |

**界面设计**：
- 列表/网格两种视图模式切换
- 列表视图：表格形式，列包括缩略图、标题、来源、大小、日期、状态

- 按日期范围筛选

**操作**：
- 双击打开文件（用系统默认播放器）
- 右键菜单：打开文件/打开所在文件夹/复制链接/重新下载/删除记录
- 批量选择 + 批量删除
- 「清空全部历史」按钮（二次确认）

#### F4.2 下载统计面板

| 项目 | 说明 |
|------|------|
| **功能描述** | 可视化展示下载数据统计 |

**统计数据**：
- 总下载次数
- 总下载大小（GB）
- 今日/本周/本月下载量
- 下载来源分布饼图（YouTube 45%、B站 30%、其他 25%）
- 格式分布饼图（MP4 60%、MP3 25%、MKV 15%）
- 每日下载趋势折线图（最近 30 天）

#### F4.3 视频转音频（后处理）

| 项目 | 说明 |
|------|------|
| **功能描述** | 下载完成后将视频转换为音频文件 |

**实现方式**：
- 下载时直接选择「仅音频」格式
- 已下载的视频，右键菜单「提取音频」
- 支持格式：MP3（128k/192k/320k）、M4A、FLAC、WAV
- 后处理使用 FFmpeg 执行
- 转换过程显示进度

#### F4.4 内置视频预览

| 项目 | 说明 |
|------|------|
| **功能描述** | 在应用内简易预览已下载的视频 |

**需求**：
- 在下载历史/已完成任务中，点击预览按钮打开内置播放器
- 使用 HTML5 `<video>` 标签实现基本播放
- 支持：播放/暂停、进度条、音量、全屏
- 支持 MP4、WebM 格式（浏览器原生支持的格式）
- MKV 等不支持的格式，提示用外部播放器打开
- 这是轻量预览功能，不追求专业播放器体验

## 附录 B：数据模型

### 下载任务 (downloads)

```typescript
interface DownloadTask {
  id: string;                  // 唯一ID
  url: string;                 // 原始URL
  title: string;               // 视频标题
  thumbnail: string;           // 缩略图URL
  author: string;              // 作者/上传者
  duration: number;            // 时长（秒）
  sourceSite: string;          // 来源域名
  
  // 下载参数
  format: string;              // 选择的格式 (mp4/mkv/mp3...)
  resolution: string;          // 选择的分辨率
  subtitleLang: string[];      // 字幕语言
  outputDir: string;           // 下载目录
  filename: string;            // 文件名
  
  // 状态
  status: TaskStatus;          // 任务状态
  progress: number;            // 0-100
  speed: string;               // 下载速度
  eta: string;                 // 预计剩余时间
  filesize: number;            // 文件大小（字节）
  downloadedSize: number;      // 已下载大小
  
  // 错误
  error: string | null;        // 错误信息
  retryCount: number;          // 重试次数
  
  // 时间戳
  createdAt: number;           // 创建时间
  startedAt: number | null;    // 开始时间
  completedAt: number | null;  // 完成时间
  
  // 输出
  filepath: string | null;     // 下载完成的文件路径
}

type TaskStatus = 
  | 'queued' 
  | 'parsing' 
  | 'downloading' 
  | 'post_processing'
  | 'completed' 
  | 'paused' 
  | 'cancelled'
  | 'parse_failed'
  | 'download_failed';
```
